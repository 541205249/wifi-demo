## 验光仪项目开发计划说明

### 一、项目概况

本项目为基于 WiFi 通信的验光仪业务应用开发，整体建设内容不仅包含页面开发，还包括业务流程梳理、基础框架搭建、设备通信接入、指令联调、提测联调及缺陷修复等完整交付环节。

本次评估口径按整个项目从启动到可提测的完整开发周期测算，不区分“已完成代码”和“待完成代码”。当前项目中已有的页面骨架、状态流、通信服务和命令配置等实现，统一计入基础框架搭建阶段。

### 二、整体开发计划

项目整体拟分为十个阶段推进：

1. 需求确认与定稿，完成功能范围、页面细节、协议边界及首版交付范围的最终确认。
2. 基础框架搭建，完成项目基础结构、主导航、页面骨架、状态流、数据模型、WiFi 通信基础能力，以及联调模式建设。
3. 工作台主流程开发，完成验光主页及核心验光操作流程落地。
4. 被测者模块开发，完成被测者新增、编辑、检索、扫码导入及与当前验光流程绑定。
5. 程序模块开发，完成程序选择、步骤展示、步骤模型及执行态联动。
6. 报告与历史模块开发，完成报告查看、历史记录、临时数据及导入导出基础能力。
7. 设备管理开发，完成设备绑定解绑、历史设备管理及通信日志能力。
8. 设置模块开发，完成公司信息、步长、语言时间及命令设置入口等配置能力。
9. 指令联调，完成页面动作与命令编码映射、编码表校验、收发回执处理及关键命令打通。
10. 提测联调与缺陷修复，完成自测、真机联调、回归验证、问题修复及提测准备。

### 三、工作量评估

按 1 人投入评估，项目总工作量为 **67 人天**。各阶段工作量如下：

| 阶段 | 工作量 |
| --- | --- |
| 需求确认与定稿 | 5 天 |
| 基础框架搭建 | 7 天 |
| 工作台主流程开发 | 15 天 |
| 被测者模块 | 5 天 |
| 程序模块 | 6 天 |
| 报告与历史模块 | 5 天 |
| 设备管理 | 2 天 |
| 设置模块 | 2 天 |
| 指令联调 | 8 天 |
| 提测联调与缺陷修复 | 12 天 |

### 四、周期评估

按单人全职投入测算，整体开发周期预计约 **13-14 周**。

其中，前期需求确认和基础框架搭建完成后，可进入主要业务模块开发阶段；中后期的关键影响项主要集中在指令联调、真机联调以及提测后的问题修复。这几个阶段对外部设备、协议资料和联调资源依赖较强，通常也是决定项目最终周期的核心因素。

### 五、重点说明

1. 本项目不是单纯的页面开发项目，核心难点在于业务流程、设备通信和指令联调的结合。
2. 当前项目采用 WiFi 通信架构，涉及前台 `Service`、TCP 连接、设备识别、通信日志和设备历史管理，联调复杂度高于普通业务应用。
3. 工作台模块是整个项目的核心，其开发量和联调量都显著高于其他业务模块。
4. 指令联调和提测联调阶段存在较强的不确定性，如设备响应差异、协议调整、现场环境问题等，都可能对周期造成影响。
5. 为保证项目可控推进，建议严格执行需求定稿后再进入大规模开发，避免在中后期因需求反复导致返工。

### 六、结论

综合评估，验光仪项目按完整交付口径测算，总工作量预计 **67 人天**，单人整体开发周期预计 **13-14 周**。建议按“需求定稿、框架搭建、业务开发、指令联调、提测修复”五个大阶段推进，并将联调阶段作为重点风险环节进行管理。
