# 验光仪页面模块梳理

## 1. 文档目的

本文基于 `app/验光仪功能说明.docx` 进行整理，目标不是复述原文，而是把需求重组为适合 App 页面开发和模块拆分的结构化清单。

整理时结合了当前工程的页面骨架，优先按以下一级模块来划分：

- 工作台 / 验光主页
- 被测者
- 报告
- 程序
- 设备与通信
- 设置
- 数据与历史

## 2. 一级模块总览

| 模块 | 建议页面形态 | 核心职责 |
| --- | --- | --- |
| 工作台 / 验光主页 | 主页面 | 承载验光主流程、视标显示、验光数据调节、镜片控制、底部检查工具 |
| 被测者 | 独立页面 / 弹窗 | 录入、检索、扫码导入被测者资料 |
| 报告 | 独立页面 | 查看和管理当前客户验光报告、打印、保存、导入、二维码分享 |
| 程序 | 列表页 + 详情配置页 | 管理验光程序模板、步骤、跳过条件、视功能配置 |
| 设备与通信 | 独立页面 | 管理验光仪、打印机、灯光控制、命令发送、连接状态 |
| 设置 | 多个设置子页 | 公司信息、云端上传、步长、时间、语言、显示选项 |
| 数据与历史 | 报告历史页 / 临时数据页 | 历史记录、临时数据、导入导出、缓存管理 |

## 3. 工作台 / 验光主页模块

### 3.1 模块定位

这是整个产品的核心页面，对应医生或验光人员的主操作台。建议作为当前 `nav_workbench` 的主页面承载。

### 3.2 页面建议拆分

| 子区域 | 建议形态 | 说明 |
| --- | --- | --- |
| 左侧功能区 | 固定按钮组 | 放置被测者、帮助、IN、打印、SHIFT、报告等快捷入口 |
| 中上验光数据区 | 数据栏 + 选中态 | 展示 SPH、CYL、AXIS、ADD、VA、棱镜等当前验光值 |
| 中部调节区 | 旋钮 / 十字键 / 步长控制 | 对选中的验光数据进行增减或切换 |
| 右侧视标区 | 视标列表 + 展示区 | 切换不同检查项目、视力表、融合/红绿/隐斜等图表 |
| 底部功能区 | 功能按钮区 | 远近切换、开灯/关灯、NPC、NPA、NRA、PRA、AC/A、AMP |
| 程序状态区 | 当前程序提示 | 展示当前选中的验光程序和步骤 |

### 3.3 左侧功能区

| 功能 | 说明 | 建议实现 |
| --- | --- | --- |
| 被测者 | 打开客户信息录入或选择弹窗 | 跳转或弹出被测者模块 |
| 帮助 | 对当前选中的视标给出说明 | 依赖当前视标上下文 |
| IN | 将当前选中的镜片或功能置入光路 | 抽象为设备命令，不绑定具体传输层 |
| 打印 | 打印当前客户验光数据 | 抽象为打印服务或设备命令 |
| SHIFT | 功能复用或切换镜片组 | 与步长、镜片组、快捷操作联动 |
| 报告 | 打开当前客户的报告页面 | 跳转报告模块 |

### 3.4 右侧视标 / 检查项目区

文档提到的右侧视标项目如下，建议统一抽象为“检查项目”：

| 检查项目 | 功能说明 |
| --- | --- |
| 隐斜视 | 隐斜视检查 |
| 带固视的隐斜视 | 带固视目标的隐斜视检查 |
| 立体视 | 双眼立体视检查 |
| 垂直重合 | 垂直方向双眼重合检测 |
| Schober | Schober 检查 |
| Worth | Worth 四点检查 |
| 固视 | 固视能力或固视状态检查 |
| 红/绿 | 红绿平衡检测 |
| 偏振红/绿 | 偏振条件下红绿检测 |
| VA表 | 常规视力表 |
| 交叉格栅 | 交叉格栅检查 |
| 散光表 | 散光方向与敏感度检查 |
| 斑点图 | 斑点图视标检查 |
| 双眼平衡 | 双眼平衡测试 |

### 3.5 验光数据调节区

文档明确提到以下验光数据项和调节规则：

| 参数 | 含义 | 步进 / 范围 |
| --- | --- | --- |
| SPH | 球镜，近视或远视度数 | ±0.25 |
| CYL | 柱镜，散光度数 | ±0.25 |
| AXS / AXIS | 轴位，散光方向 | 步进 ±25，范围 0~180 |
| ADD | 下加光 / 近用附加度 | ±0.25 |
| VA | 矫正视力 | 随视力表范围变化自动联动 |
| X / Y 或棱镜值 | 棱镜水平 / 垂直值 | 受坐标模式影响 |

配套控制规则：

- 支持右镜片、左镜片、双眼镜片切换。
- 支持通过选中上方验光数据，再用圆盘顺时针/逆时针调整数值。
- 支持通过圆盘的上下左右切换同类视标或同组检查项。
- 支持柱镜模式切换：
  - `C-`：柱镜允许负值
  - `C+`：柱镜只允许正值
- 支持棱镜显示模式切换：
  - 直角坐标模式：按 X/Y 展示
  - 极坐标模式：按总棱镜度 R 与角度 θ 展示

### 3.6 镜片 / 视标联动能力

文档中的镜片与视标联动能力建议单独封装：

- 右镜片显示
- 左镜片显示
- 双眼镜片显示
- 视力表与镜片联动开关
- 视标方向切换：
  - 横排显示
  - 竖排显示
  - 单个视标显示

### 3.7 底部检查工具区

| 功能 | 说明 | 调节规则 |
| --- | --- | --- |
| 远距 / 近距切换 | 切换当前测试距离 | 主页面底部按钮切换 |
| 开灯 | 控制设备开灯 | 抽象为设备命令 |
| 关灯 | 控制设备关灯 | 抽象为设备命令 |
| NPC | 近点集合 | 数值大于 0，步进 1，可自动按距离计算 |
| NPA | 近点调节 | 数值大于 0，步进 1，可自动按距离计算 |
| NRA | 负相对调节 | 步进 0.25，支持模糊/恢复记录 |
| PRA | 正相对调节 | 小于 0，步进 0.25，支持模糊/恢复记录 |
| AC/A | 调节性集合 / 调节比 | BI 步进 0.5，支持首次对齐 / 再次对齐 |
| AMP | 调节幅度 | 支持左右镜片切换，步进 1 |

### 3.8 工作台附属子页 / 子弹窗

文档提到了若干工作台相关附页，建议归档为工作台附属功能：

- 图表界面
- 图标新增 / 修改界面
- 设备连接界面
- 步长界面
- 时间界面

其中“设备连接界面”应统一表达 WiFi 服务、设备发现、设备绑定和外设接入，不再把传输层写死为某一种通信方式。

## 6. 程序模块

### 6.1 模块定位

负责管理验光程序模板、程序步骤和各步骤的参数配置。该模块是“验光流程引擎”的管理后台。

### 6.2 首页选择能力

文档明确提到三种验光程序可在首页直接选择，选择完成后在首页左下角展示当前程序。

因此建议程序模块至少包含：

- 程序列表
- 程序详情
- 程序步骤编辑
- 当前程序状态展示

### 6.3 程序类型

根据文档，可整理为三类程序：

| 程序类型 | 说明 |
| --- | --- |
| 常规验光 | 标准屈光检查流程 |
| 视功能快速筛查 | 精简版视功能检查流程 |
| 全套视功能检查 | 全量视功能检查流程 |

### 6.4 常规验光步骤

文档列出的常规验光步骤如下：

1. 雾视

5. 首次 MPMVA（右眼）
6. 首次红绿检测（右眼）
7. 精调散光轴位（右眼）
8. 精调散光度数（右眼）
9. 二次红绿检测（右眼）
10. 单眼 MPMVA（右眼）
11. 去雾视（左眼）
12. 粗调散光轴位（左眼）
13. 粗调散光度数（左眼）
14. 首次 MPMVA（左眼）
15. 首次红绿检测（左眼）
16. 精调散光轴位（左眼）
17. 精调散光度数（左眼）
18. 二次红绿检测（左眼）
19. 单眼 MPMVA（左眼）
20. 双眼最佳视力检测
21. 双眼平衡测试
22. 双眼 MPMVA
23. 试戴

### 6.5 视功能快速筛查步骤

文档提到的快速筛查步骤包括：

- 双眼融合测试
- NRA：负相对调节
- PRA：正相对调节
- 远眼位
- 近眼位

### 6.6 全套视功能检查步骤

文档提到的全套视功能检查步骤包括：

- 双眼融合测试
- 立体视测试
- 远眼位
- 近眼位
- AC/A（梯度法）
- NRA：负相对调节
- BCC 检查
- PRA：正相对调节
- AMP 检查
- 远距水平融像
- 近距水平融像

### 6.7 程序步骤可配置项

文档中反复出现的步骤配置字段如下，建议沉淀为统一的步骤模型：

| 字段 | 可选值 |
| --- | --- |
| AR | LM、AR、SubJ、Un-aided、Final |
| 远距 | 近距、远距 |
| B | L、R、B |
| 参数类型 | SPH、CYL、AXIS、VA、ADD、X、Y |
| 导入数据 | 否、是 |
| 雾视 | 空值、+0.25、+0.50、+0.75、+1.00、+1.50、+3.00 |
| 视近灯 | 关、开 |
| 视功能 | 根据步骤不同有不同候选值 |
| 跳过字段名 | SPH、CYL、AXIS、VA、ADD、X、Y、无 |
| 跳过比较符 | =、>、>=、<、<= |

### 6.8 程序模块实现建议

- 程序模板与步骤配置建议做成可持久化的数据模型。
- 步骤配置页建议采用列表 + 详情编辑页模式。
- 程序执行态与程序配置态分离。
- 程序步骤未来很可能要和设备命令编码表联动。

## 11. 开发时需要特别注意的点

### 11.1 文档中有不少图标型按钮

提取文本时，部分“加号 / 减号 / 左右箭头 / 图标按钮”的具体字符没有完整提取出来，文档里多处出现“点击图中与”“点击按钮”等表述。实际开发时应回看原图确认：

- 图标样式
- 点击区域
- 是否短按 / 长按
- 是否有选中态 / 禁用态

### 11.2 通信能力不要写死成具体传输层

本项目当前是纯 WiFi 架构，因此设备控制层应统一抽象，UI 只表达“连接设备 / 控制设备 / 打印 / 开关灯 / 置入光路”，不直接绑死具体实现。

### 11.3 程序模块和命令编码表未来会强关联

程序步骤、工作台按钮、底部检查项，后续都可能对应设备命令编码，因此当前建模时建议提前预留：

- 页面功能 ID
- 命令编码 ID
- 步骤模板 ID
- 视标 / 检查项目 ID

- 工作台主页面骨架

- 设备通信基础层

### 第二阶段

- 报告模块

- 程序步骤编辑器

## 13. 一句话结论

如果按页面开发来拆，这个产品的核心不是单纯的“验光主页”，而是：

**以工作台为中心，向外扩展被测者、程序、报告、设备、设置五个一级业务模块，再由数据与历史模块做统一承接。**
